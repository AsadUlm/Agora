# [SW중심대학] 2026-1 캡스톤디자인 교과목 팀별 과제신청서식 (학생작성용)

---

## 1. 과제의 개요

현대 사회는 정보 과부하 속에서 중요한 의사결정을 내려야 하는 상황이 빈번하게 발생한다. 개인이 단일 AI 모델에 의존할 경우, 해당 모델의 편향이나 지식 한계로 인해 일면적인 판단에 그칠 위험이 있다. **AGORA**는 이러한 한계를 극복하기 위해 설계된 **다중 에이전트(Multi-Agent) AI 토론 기반 의사결정 지원 플랫폼**이다.

AGORA는 사용자가 하나의 질문 혹은 주제를 입력하면, 서로 다른 역할과 추론 방식을 가진 복수의 AI 에이전트(예: 분석가, 비평가, 낙관론자, 리스크 분석가)가 구조화된 토론을 수행하고, 중립적 사회자(Moderator Agent) 역할을 맡은 Claude Sonnet 4.5가 각 라운드의 합의점과 쟁점을 분석하여 사용자에게 제공하는 시스템이다.

토론은 총 3라운드로 구성된다:
- **Round 1 (Opening Statements):** 각 에이전트가 독립적으로 초기 입장을 제시
- **Round 2 (Cross Examination):** 충돌하는 주장 쌍이 도전-응답-재반박 형태로 심화 토론
- **Round 3 (Final Synthesis):** 각 에이전트의 최종 입장 변화와 종합 결론 제시

사용자는 자신의 문서(PDF, 논문, 보고서 등)를 업로드하여 에이전트가 RAG(Retrieval-Augmented Generation) 기반으로 내부 지식을 근거로 주장을 펼치게 할 수 있으며, 각 에이전트에게 서로 다른 AI 모델(GPT-5, Gemini 2.5 Flash, DeepSeek V3.2, Claude Haiku 4.5, Grok 4.1 Fast 등 8종)을 자유롭게 배정할 수 있다.

AGORA는 단순한 챗봇이나 정보 검색 도구가 아닌, 복잡한 문제를 다양한 시각에서 구조적으로 조망하게 하는 **AI 기반 집단 지성 의사결정 인터페이스**이다.

## 2. 과제의 목표

### 최종 결과물

| 구분 | 내용 |
|------|------|
| 웹 플랫폼 | 다중 에이전트 AI 토론 기반 의사결정 지원 서비스 (프론트엔드 + 백엔드 + AI 오케스트레이션 포함) |
| 핵심 기능 | 3단계 구조화 토론 (Opening → Cross Examination → Final Synthesis), RAG 연동, 실시간 Moderator 패널 |
| 배포 형태 | 웹 애플리케이션 (클라우드 배포), 사용자 계정 관리 및 세션 이력 지원 |

### 정량적 목표

- 토론 세션 수: 프로젝트 기간(2026년 3월 23일 ~ 6월 11일, 80일) 동안 최소 2,320 누적 세션 처리 가능한 시스템 안정성 달성
- 에이전트 수: 최대 4개 AI 에이전트 동시 운용
- 지원 모델 수: 8종 이상의 이종 LLM 통합 (GPT-5, GPT-4.1 Mini, Gemini 2.5 Flash, Claude Haiku 4.5, DeepSeek V3.2, Kimi K2.5, Grok 4.1 Fast, Grok 4 Thinking)
- 인프라 비용: OpenRouter 기반 과금 제어를 통해 프로젝트 전체 인프라 비용 600,000원 이내 유지
- 백로그 완료율: 정의된 스프린트 Product Backlog 기준 90% 이상 태스크 완료

### 정성적 목표

- 단일 AI 모델 의존 대비 다양한 관점 제공을 통한 의사결정 품질 향상
- RAG 기반 근거 인용으로 주장의 출처 투명성 확보
- 복잡한 다중 에이전트 토론을 직관적인 UI/UX로 시각화하여 접근성 향상
- 사용자가 AI 토론 구조를 이해하고 능동적으로 제어할 수 있는 인터렉션 설계

### 산업체 연계 전략 및 결과물 활용 방안

AGORA는 법률 검토, 경영 전략 수립, 학술 연구 방향 결정, 정책 분석 등 다수의 이해관계자가 복잡한 쟁점을 검토해야 하는 분야에 직접 적용 가능하다. 분야별 RAG 지식 베이스를 커스터마이징하는 B2B SaaS 형태의 서비스 확장이 가능하며, 교육 분야에서는 AI 토론 학습 도구로도 활용될 수 있다.

## 3. 결과물에 대한 기대효과 및 활용방안

### 유형적 효과

- **다중 LLM 오케스트레이션 프레임워크:** 이종 LLM을 하나의 구조화된 토론 파이프라인으로 통합하는 실용적 구현체로, 오픈소스 공개 또는 기술 이전 가능
- **RAG 기반 에이전트 지식 연동 시스템:** 사용자 문서를 실시간으로 파싱·임베딩·검색하는 라이브러리 수준의 모듈
- **비용 최적화 모델:** OpenRouter를 활용한 다중 LLM 정밀 비용 계산 및 예산 제어 방법론 (80일 2,320 세션 기준 600,000원 이내 운용 실증)
- **특허/실용신안 출원 가능성:** 다중 에이전트 구조화 토론 및 Moderator 기반 실시간 쟁점 분석 메커니즘의 신규성 존재

### 무형적 효과

- AI 활용 의사결정 과정의 투명성 및 신뢰도 제고
- 개발자 역량: 풀스택 AI 애플리케이션 설계 및 LLM 오케스트레이션 실무 역량 확보
- 오픈AI·앤트로픽·구글 등 다양한 API 생태계 연동 경험 축적

### 상품화 가능성 및 응용 분야

| 활용 분야 | 구체적 적용 방식 |
|-----------|----------------|
| 기업 의사결정 지원 | 경영진의 전략적 판단 시 다각도 AI 의견 수렴 도구 |
| 법률·정책 분석 | 상반된 법리 해석을 에이전트 간 토론 형식으로 정리 |
| 학술 연구 지원 | 논문 검토 시 찬반 논거 자동 구조화 |
| 교육 플랫폼 | AI 토론 시뮬레이션 기반 비판적 사고 훈련 도구 |
| 개인 의사결정 | 진로, 투자, 정책 선택 등 개인 레벨의 복잡한 선택 지원 |

## 4. 수행방법

개발 환경 및 기술 스택

- 프론트엔드: React, TypeScript, React Flow (시각화)
- 백엔드: Node.js / Python FastAPI
- AI 오케스트레이션: OpenRouter API (다중 LLM 통합), LangChain 또는 자체 오케스트레이터
- RAG 파이프라인: 문서 파싱(PDF/DOCX/HWP), 텍스트 청킹, 임베딩 생성, 벡터 검색
- 데이터베이스: PostgreSQL (세션 및 메타데이터), 벡터 DB (임베딩 인덱스)
- 배포: 클라우드(GCP / AWS), 컨테이너 기반 배포
- 비용 관리: OpenRouter per-request cost 필드 추적, BYOK(Gemini 무료 키) 활용

수행 절차

1단계. 기획 및 설계 (1~2주)
사용자 스토리 기반 Product Backlog를 확정하고, 에이전트 오케스트레이션 흐름 및 DB 스키마를 포함한 시스템 아키텍처를 설계한다. UI/UX 와이어프레임 및 프로토타입도 이 단계에서 작성한다.

2단계. 핵심 기능 구현 (3~8주)
회원가입/로그인 및 세션 관리를 구현하고, 에이전트 설정 UI와 백엔드 API(모델 선택, 역할 템플릿, 최대 4 에이전트)를 개발한다. 토론 3라운드 오케스트레이션 로직(병렬 실행, 스트리밍, 실패 재시도)과 Moderator 패널 실시간 연동도 이 단계에서 구현한다.

3단계. RAG 및 고급 기능 구현 (5~9주)
문서 업로드에서 파싱, 청킹, 임베딩, 검색까지 이어지는 RAG 파이프라인을 구현한다. 에이전트별 지식 베이스 접근 권한 설정 기능과 근거 출처 인용(Citation) UI도 함께 개발한다.

4단계. 통합 및 검증 (9~11주)
에이전트 실패, 빈 문서, 중복 업로드 등 예외 시나리오를 검증한다. 비용 시나리오(Baseline / Scenario A~C)별 실증 테스트를 수행하고 성능 및 응답 속도를 최적화한다.

5단계. 최종 점검 및 발표 준비 (12주)
전체 기능 통합 테스트 및 버그를 수정하고, 서비스를 배포한 후 데모 시나리오를 구성한다. 결과 보고서 및 최종 발표 자료를 준비한다.
